The problem

Agents can act now. Can you trust them to?

The moment an AI agent can move money, write to production, or email a customer, two questions decide whether you can actually ship it — and most teams can't answer either.

01

Can you stop the catastrophic action before it happens — the $1M refund, the dropped table, the email to every customer?

02

Can you prove, afterward, exactly what it did — and that every action was authorized?

What it does

Gate it. Prove it. Drop it in.

↯Gate

Every action is checked against your policy before it runs — allow, deny, or wait for a human. The dangerous action never executes.

✓Prove

Every action is sealed into a tamper-evident hash chain. Edit, delete, or reorder the log and verify catches it — independently, in one command.

{ }Integrate

Works with MCP, LangChain, and OpenAI/Anthropic tool calls in ~5 lines. Secrets are redacted by default. Zero dependencies.

from colorless import Colorless

cl = Colorless("agent.jsonl", on_approval=ping_slack)
cl.deny("delete_account")                 # never, full stop
cl.require_approval("refund", when=lambda a: a["args"]["amount"] > 100)

@cl.guard
def refund(amount, to):
    return payments.refund(amount, to)

refund(amount=80, to="cust_12")     # runs — sealed in the chain
refund(amount=5000, to="cust_12")   # blocked — waits for a human

cl.verify()  # {"ok": True, ...} — proof nothing was altered
